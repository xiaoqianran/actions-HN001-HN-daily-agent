# 摘要提示词模板（可选）

你可以将以下任意模板内容复制到 GitHub Actions Variable `SUMMARY_PROMPT_TEMPLATE` 中使用。

> 每个模板都**必须包含** `{title}` 和 `{content}` 两个占位符。

---

## 模板 1：通用技术摘要（默认推荐）

```text
请阅读以下内容并给出中文摘要。
标题：{title}
正文：{content}

要求：
1. 先用 1 句话给出核心结论；
2. 再列出 2-4 条关键点；
3. 语言简洁，适合技术读者快速浏览。
```

## 模板 2：面向决策者（价值导向）

```text
请基于以下文章生成中文决策摘要。
标题：{title}
内容：{content}

输出格式：
- 一句话结论
- 业务/技术价值（2-3 点）
- 潜在风险或限制（1-2 点）
- 是否值得进一步关注：是/否 + 理由
```

## 模板 3：工程实践导向

```text
请将以下内容总结为工程团队可执行的信息。
标题：{title}
正文：{content}

请输出：
1) 这篇内容解决了什么问题；
2) 涉及的关键技术或方案（2-5 条）；
3) 可落地实践建议（1-3 条）；
4) 一句话总结。
```

## 模板 4：学习笔记风格

```text
请把下面文章整理成中文学习笔记。
标题：{title}
内容：{content}

要求：
- 先给出「核心观点」；
- 再给出「知识点清单」（3-6 条）；
- 最后给出「我可以立即尝试的一件事」。
```

## 模板 5：极简快讯（短文本）

```text
请把以下内容压缩成一条中文快讯。
标题：{title}
正文：{content}

要求：
- 总字数控制在 80-120 字；
- 保留最关键的信息；
- 不要使用空泛表述。
```
